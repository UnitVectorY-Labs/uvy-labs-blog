We're excited to announce the release of gogitup v0.1.4, published on March 2, 2026. This update brings two significant quality-of-life improvements that make installing and configuring Go tools even easier.

## What's New

### One-Command Installation

The biggest addition in v0.1.4 is the new `install` command. Previously, installing a Go tool from GitHub required finding the latest release tag and running `go install` manually. Now you can do it all in one step:

```bash
gogitup install UnitVectorY-Labs/gogitup
```

This single command fetches the latest release from GitHub, installs the binary using `go install`, and automatically registers it for version tracking. No need to look up release tags or remember to register your tools afterward.

### Custom Go Proxy Configuration

For teams working behind corporate proxies or with custom module registries, v0.1.4 adds support for configuring a custom GOPROXY through gogitup's configuration file:

```yaml
# ~/.gogitup
apps:
  - name: ghorgsync
  - name: bulkfilepr
goproxy: "https://proxy.example.com,direct"
```

When configured, this setting is automatically applied to all `go install` and `update` operations, ensuring consistent proxy behavior across your tooling without relying solely on environment variables.

## Why It Matters

These changes address two common pain points for developers who rely on Go-based CLI tools:

**Simplified onboarding**: The `install` command removes friction from getting started with new tools. Instead of navigating GitHub releases, determining version tags, and managing registration separately, you now have a single workflow that handles everything. This is especially helpful when setting up new development environments or sharing tool installation instructions with teammates.

**Enterprise readiness**: The configurable proxy support means gogitup can now be used in environments where direct access to the public Go module proxy isn't available or where organizations require all module downloads to go through approved infrastructure. Combined with gogitup's existing features, this makes it a viable option for corporate development teams.

Both features maintain complete backward compatibility—your existing configuration files continue to work without any changes required.

## Getting Started

### Installing gogitup v0.1.4

If you're new to gogitup, install the latest version:

```bash
go install github.com/UnitVectorY-Labs/gogitup@v0.1.4
```

Binary releases are also available for macOS (Intel and Apple Silicon), Linux (x86_64, ARM64, 386), and Windows from the [release page](https://github.com/UnitVectorY-Labs/gogitup/releases/tag/v0.1.4).

### Trying the New Features

Once installed, give the `install` command a try:

```bash
gogitup install <owner/repo>
# Example: gogitup install UnitVectorY-Labs/gogitup
```

To use the new proxy configuration, add the `goproxy` field to your `~/.gogitup` config file and restart your terminal.
